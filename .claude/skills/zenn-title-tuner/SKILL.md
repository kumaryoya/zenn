# Zenn Title Tuner

Zenn 記事のタイトルを、同 topic でトレンドに入っている記事の傾向を踏まえて書き直す案を提示するスキル。

## 入力

ユーザーが渡す引数 = 対象記事ファイルのパス（リポジトリルートからの相対パス、または絶対パス）。
未指定の場合は、IDE で開いている `articles/*.md` があればそれを対象候補として確認する。なければユーザーに聞く。

## 手順

1. **記事を読む**
   - Read で対象ファイルを開き、frontmatter から `title` と `topics` を抽出する。
   - 本文の冒頭〜「## はじめに」あたりまでを読み、記事の主題と読者像をざっくり把握する（タイトル案の根拠に使う）。

2. **同 topic のトレンドを取得**
   - Zenn の HTML ページは記事カードをクライアント側でレンダリングするため WebFetch ではタイトル一覧が取れない。**JSON API を直接叩く**こと。
   - エンドポイント: `https://zenn.dev/api/articles?topicname=<topic>&order=daily&count=20`
   - 各 topic に対して **並列で** Bash の `curl` を実行する（1 メッセージ内で複数 tool call）。topic は frontmatter のとおり小文字・スペースなしでそのまま URL に入れて良い。
   - 取得した JSON はそのまま貼ると長いので、Bash の中で `python3 -c` で `articles[].title` と `liked_count` だけ抽出する。例:
     ```bash
     curl -s "https://zenn.dev/api/articles?topicname=aws&order=daily&count=20" \
       | python3 -c "import sys,json; d=json.load(sys.stdin); [print(f'{a[\"liked_count\"]:>4}♥ {a[\"title\"]}') for a in d['articles']]"
     ```
   - 取得失敗・該当 topic ページ無しの場合はその旨をユーザーに伝え、残った topic だけで続行する。

3. **トレンドの型を抽出**
   - 集めたタイトル群から、再現性のあるパターンを 3〜5 個メモする。例:
     - 数字を入れる（「3 つの〜」「N 倍」）
     - 対象読者を明示（「初心者向け」「現場の SRE が〜」）
     - ベネフィット先出し（「〜で本番障害を防ぐ」）
     - 角括弧やラベル接頭辞（「【AWS】」「[実例]」）
     - 問い・逆説・煽り（「〜は本当に必要か」「やめた話」）
   - そのうち、**今回の記事の内容と整合する型だけ**を採用する。記事内容と乖離した煽りは禁止（誇大タイトルは Zenn コミュニティで嫌われる）。

4. **タイトル案を提示**
   - 現タイトルを冒頭に再掲し、改題案を **3〜5 案**、それぞれ次の形式で出す:
     ```
     1. <タイトル案>
        - 採用した型: <例: 数字 + 対象読者明示>
        - 根拠トレンド例: <参考にした既存タイトル 1 件>
     ```
   - 案は 35 文字以内を目安にする（Zenn 一覧でのトリミング回避）。
   - 既存の絵文字（emoji frontmatter）はタイトル文には含めない。
   - 全案出した後に、**1 行のおすすめと理由** を添える。

5. **書き換えはしない**
   - frontmatter の `title` を勝手に書き換えない。ユーザーが「これで差し替えて」と言ったら Edit する。

## 出力スタイル

- 日本語、簡潔に。
- トレンド分析の生データ（取得した全タイトル）は出力に貼らない。要約だけ。
- 不確実な部分（ページ構造の変化でトレンド判定が曖昧、など）は素直に書く。
