## GitHub Copilot とは

GitHub Copilot とは、GitHub が提供している AI コーディングアシスタントです。

コード補完やコードレビュー、コーディングエージェントなど、様々な機能が搭載されており、開発者の生産性を向上させることができます。

GitHub Copilot は、GitHub や VS Code、Neovim など、様々な開発環境で使用することができます。

https://github.com/features/copilot?locale=ja

https://docs.github.com/ja/copilot

## GitHub Copilot のプラン

GitHub Copilot のプランには、「個人向け」と「ビジネス向け」があり、「個人向け」には3つのプランが、「ビジネス向け」には2つのプランがあります。

プランによって、使える機能や LLM の種類が異なります。

また、グレードが上のプランでは、プレビュー段階の機能も使用できるようになります。

コストと機能のバランスを考慮して、適切なプランを契約し、GitHub Copilot を使ってみてください。

https://github.com/features/copilot/plans?locale=ja

## 私の GitHub Copilot の使い方

### 1. コード補完

コード補完は、GitHub Copilot の最も基本的な機能で、私は契約当初からずっと使っています。

私は普段、VS Code でコードを書いています。

コードを書いていると、GitHub Copilot が自動的に続きのコードを提案してくれます。

提案されたコードは、`Tab` キーで受け入れることができ、`Esc` キーで拒否することができます。

メソッド名からコードを推測して提案してたり、コメントからコードを生成して提案してくれたりします。

私のこの機能の使い方に関しては、特に説明すべきことはありません。

https://docs.github.com/ja/copilot/get-started/features#inline-suggestions

### 2. Agent モード、Ask モード

VS Code では、サイドバーにチャット画面を開くことで、GitHub Copilot と対話することができます。

モードと LLM を選択することで、GitHub Copilot に質問したり、コードの説明を求めたり、コードの生成を依頼したりすることができます。

:::message
- モード
  - Agent モード
  - Plan モード
  - Ask モード
  - Edit モード

- LLM
  - GPT シリーズ
  - Claude シリーズ
  - Gemini シリーズ
:::

#### Agent モード

このモードは、GitHub Copilot に指示を与えることで、自律的に作業を進めてくれるモードです。

プロジェクト全体を考慮したうえで、複数のステップにわたるアクションを実行してくれます。

私は、以下のような場面で Agent モードを活用しています。

- **新機能の実装**：「〇〇機能を実装して」と指示すると、必要なファイルを作成・修正してくれる
- **バグ修正**：「〇〇のバグを修正して」と指示すると、問題を特定して修正してくれる
- **リファクタリング**：「〇〇部分をリファクタリングして」と指示すると、複数のファイルにまたがるリファクタリングを実行してくれる

まだ現時点では、作成・修正したコードをそのまま提出することはできませんが、最後に自分で微修正すれば問題ないレベルのコードを生成してくれるので、かなり開発の効率化に役立っています。

このように、複数のファイルにまたがる作業が必要な場合は、Agent モードを使うようにしています。

https://docs.github.com/ja/copilot/get-started/features#agent-mode

現時点では、「Agent モード」、「Planモード

GitHub Copilot のチャット機能を使うと、自然言語で質問したり、コードの説明を求めたりすることができます。


私は、以下のような場面でチャット機能を活用しています。

- **コードの説明**: 理解できないコードがあるときに、そのコードを選択して「このコードを説明して」と聞く
- **エラーの解決**: エラーメッセージをコピーして、「このエラーの解決方法を教えて」と聞く
- **コードの生成**: 「〇〇を実現するコードを書いて」と指示して、コードを生成してもらう
- **リファクタリング**: 既存のコードを選択して、「このコードをリファクタリングして」と依頼する
- **テストコードの生成**: 関数を選択して、「このテストコードを書いて」と依頼する

#### チャットの例

例えば、以下のようにコードを選択してチャットで質問すると、コードの説明を返してくれます。

```
// 選択したコード
const result = array.reduce((acc, cur) => acc + cur, 0);

// チャットでの質問
「このコードを説明して」

// GitHub Copilotの回答
「このコードは、配列の全要素の合計を計算しています。reduceメソッドを使用して..."
```

また、エラーが発生したときに、エラーメッセージをコピーしてチャットに貼り付けると、解決方法を提案してくれます。

### 3. コーディングエージェント

コーディングエージェントは、GitHub Copilot の中でも特に強力な機能です。

この機能を使うと、GitHub Copilot が自動的にコードを書いたり、複数のファイルを編集したり、プルリクエストを作成したりしてくれます。

VS Code では、チャットで `#github-pull-request_copilot-coding-agent` を使用するか、GitHub の Issue に Copilot を割り当てることで利用できます。

私は、以下のような場面でコーディングエージェントを活用しています。

- **新機能の実装**: 「〇〇機能を実装して」と指示すると、必要なファイルを作成・編集してくれる
- **バグ修正**: 「〇〇のバグを修正して」と指示すると、問題を特定して修正してくれる
- **大規模なリファクタリング**: 複数のファイルにまたがるリファクタリングを依頼する
- **ドキュメントの生成**: README やコメントの追加を依頼する

#### コーディングエージェントの使い方

**VS Code での使い方**:

1. チャットパネルを開く
2. `#github-pull-request_copilot-coding-agent` を入力
3. 実装してほしい内容を指示
4. GitHub Copilot が自動的にブランチを作成し、コードを編集し、プルリクエストを作成

**GitHub での使い方**:

1. Issue を作成
2. Issue の右側のメニューから「Copilot」を選択
3. 「Assign to Copilot」をクリック
4. GitHub Copilot が自動的にブランチを作成し、コードを編集し、プルリクエストを作成

コーディングエージェントは、時間のかかる作業を自動化してくれるので、非常に便利です。

### 4. コードレビュー

GitHub Copilot には、プルリクエストのコードを自動的にレビューしてくれる機能もあります。

この機能を使うと、GitHub Copilot がコードの問題点や改善点を指摘してくれるので、コードの品質を向上させることができます。

私は、プルリクエストを作成したら、まず GitHub Copilot にレビューしてもらい、指摘された点を修正してから、人間のレビュアーにレビューを依頼するようにしています。

#### コードレビューの使い方

1. GitHub でプルリクエストを開く
2. プルリクエストの右側のメニューから「Reviewers」を選択
3. 「Copilot」を選択
4. GitHub Copilot が自動的にコードをレビューし、コメントを追加

#### コードレビューで指摘される内容の例

- **パフォーマンスの問題**: 「このループは最適化できます」
- **セキュリティの問題**: 「このコードには XSS の脆弱性があります」
- **可読性の問題**: 「この変数名はより明確にできます」
- **ベストプラクティスの提案**: 「この処理は〇〇パターンを使うとより良いです」
- **バグの可能性**: 「この条件では null チェックが必要です」

GitHub Copilot のコードレビューは、人間のレビュアーが気づかないような細かい点も指摘してくれるので、コードの品質向上に役立ちます。
