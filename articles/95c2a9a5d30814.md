## はじめに

AWS のマネコンにログインする際に MFA を導入するのは一般的です。

しかし、認証情報の漏洩や不正アクセスに備えて、IP 制限を併用することでさらにセキュリティを強化できます。

本記事では、AWS のマネコンでの操作に IP 制限をかける方法について解説します。

少しでも参考になれば幸いです。

## IP 制限の仕組み

AWS のマネコンへのログイン自体を IP 制限することはできません。

今回の方法は、特定の IP アドレス以外からのすべてのリクエストを拒否するというアプローチです。

IAM ユーザーに設定するパターンと、IAM Identity Center に設定するパターンの 2 つを紹介します。

## 設定方法

### IP アドレスを確認する

まず、許可リストに追加する自分の IP アドレスを確認しておきます。

以下のようなサイトで確認できます。

https://rakko.tools/tools/2

### IAM ユーザー

#### ポリシーの作成

「IAM」>「ポリシー」>「ポリシーの作成」に進みます。

![](https://storage.googleapis.com/zenn-user-upload/5abbd5b529cd-20260320.jpg =1000x)

ポリシーエディタに以下の JSON を入力し、適当なポリシー名を付けて作成します。

```json
{
	"Version": "2012-10-17",
	"Statement": {
		"Effect": "Deny",
		"Action": "*",
		"Resource": "*",
		"Condition": {
			"NotIpAddress": {
				"aws:SourceIp": [
					"先ほど確認したIPアドレス"
				]
			}
		}
	}
}
```

#### ポリシーをアタッチする

「IAM」>「ユーザー」に進み、IP 制限を適用したいユーザーを選択します。

「許可ポリシー」>「許可を追加」をクリックします。

![](https://storage.googleapis.com/zenn-user-upload/b7ec660d160a-20260320.jpg =1000x)

「許可のオプション」は「ポリシーを直接アタッチする」を選択します。

「許可ポリシー」は先ほど作成したポリシーを選択します。

「許可を追加」をクリックして完了です。

### IAM Identity Center

#### インラインポリシーを編集する

「IAM Identity Center」>「許可セット」に進み、IP 制限を適用したい許可セットを選択します。

「インラインポリシー」の「編集」をクリックします。

![](https://storage.googleapis.com/zenn-user-upload/fd4eea240344-20260320.jpg =1000x)

以下の JSON を入力し、保存して完了です。

```json
{
	"Version": "2012-10-17",
	"Statement": {
		"Effect": "Deny",
		"Action": "*",
		"Resource": "*",
		"Condition": {
			"NotIpAddress": {
				"aws:SourceIp": [
					"先ほど確認したIPアドレス"
				]
			}
		}
	}
}
```

### 動作確認

どちらのパターンでも、以下の観点で動作確認を行います。

1. 許可した IP アドレスからマネコンにアクセスし、リソースの操作ができることを確認する
2. 許可していない IP アドレスからアクセスし、操作が拒否されることを確認する

:::message
許可していない IP からアクセスした場合、マネコンにはサインインできますが、各サービスの画面で **「You are not authorized to perform this operation.」** などのエラーが表示されます。サインイン自体は認証（Authentication）の仕組みで行われるため、今回の認可（Authorization）レベルの制限では防げません。
:::

## まとめ

- IAM ポリシーの `aws:SourceIp` 条件キーで、特定の IP アドレスからのみ操作を許可できる
- `Deny` + `NotIpAddress` の組み合わせで、許可リスト外からの操作を拒否する
- `aws:ViaAWSService` を設定して、AWS 内部の呼び出しが拒否されないようにする
- IAM ユーザーの場合はポリシーを直接アタッチ、IAM Identity Center の場合はアクセス許可セットに設定する
